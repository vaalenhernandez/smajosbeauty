# Asesoría de marca personal — @smajosbeauty

Página web interactiva con la asesoría completa de marca personal, crecimiento y monetización para María José (@smajosbeauty): diagnóstico, bio optimizada, plan de acción, plan de contenido, ideas + guiones, monetización, prospección de marcas, análisis de portafolio, identidad de marca y voz/comunicación.

Incluye navegación por secciones, checklists interactivos con progreso guardado en el navegador, botones para copiar bios/guiones/mensajes, y una paleta visual propia (vino, dorado, blanco hueso).

## Cómo verla online (GitHub Pages)

1. Sube este repositorio a GitHub (ver pasos abajo).
2. En el repositorio, ve a **Settings → Pages**.
3. En "Branch", selecciona `main` (o `master`) y la carpeta `/root`, luego **Save**.
4. GitHub te dará una URL como `https://tu-usuario.github.io/nombre-del-repo/` — ábrela y ya está publicada.

## Cómo subirlo a GitHub (paso a paso)

Si nunca has usado GitHub, la forma más fácil es sin usar la terminal:

1. Crea una cuenta en [github.com](https://github.com) si no tienes una.
2. Haz clic en **New repository** (botón verde arriba a la derecha).
3. Ponle un nombre, por ejemplo `asesoria-smajosbeauty`, y créalo como **público**.
4. Dentro del repositorio, haz clic en **uploading an existing file**.
5. Arrastra los archivos `index.html` y `README.md` de esta carpeta.
6. Haz clic en **Commit changes**.
7. Sigue los pasos de "Cómo verla online" arriba para activar GitHub Pages.

### Si prefieres usar la terminal

```bash
git init
git add index.html README.md
git commit -m "Asesoría de marca personal @smajosbeauty"
git branch -M main
git remote add origin https://github.com/TU-USUARIO/NOMBRE-DEL-REPO.git
git push -u origin main
```

## Archivos

- `index.html` — la página completa (todo el código está en un solo archivo, no necesita nada más).
- `README.md` — este archivo.

## Notas

- El progreso de las casillas y las secciones marcadas como revisadas se guarda en el navegador de cada persona que la abre (no es compartido entre dispositivos).
- Los botones "Copiar" funcionan tanto abriendo el archivo directo desde tu computador como publicado en GitHub Pages.
